# Sapience AI — Windows Run Guide

Complete setup instructions for running Sapience AI on Windows 11.
No Docker required.

---

## Prerequisites

- Python 3.11 or 3.12 (download from python.org — check "Add to PATH")
- Git (optional, for cloning)
- A Supabase account (free tier works fine)

---

## Part 1: Backend Setup

### Step 1 — Navigate to the backend folder

```powershell
cd C:\Users\WALTON\Desktop\Sapience-AI\backend
```

### Step 2 — Create virtual environment

```powershell
python -m venv .venv
```

### Step 3 — Activate virtual environment

```powershell
.venv\Scripts\activate
```

You will see `(.venv)` in your prompt. This keeps dependencies isolated.

### Step 4 — Install dependencies

```powershell
pip install -r requirements.txt
```

### Step 5 — Create your .env file

```powershell
copy .env.example .env
```

Then open `.env` in Notepad and fill in:

```
APP_NAME=Sapience AI Backend
ENVIRONMENT=development
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_KEY=your_supabase_anon_or_service_role_key
```

Find these values in your Supabase project:
**Project Settings → API → Project URL and anon/public key**

### Step 6 — Start the backend

```powershell
uvicorn app.main:app --reload --workers 1
```

Use `--workers 1` to prevent loading heavy models multiple times on low-RAM machines.

The server starts at: http://127.0.0.1:8000

Visit http://127.0.0.1:8000/docs to see the interactive API.

> If it fails with a compiler error, download a pre-built wheel from:

## Part 3: Configure Local LLM (Optional)

If you want Bengali advice generated by a local AI model:

### Step 1 — Download a GGUF model

Recommended (4 GB RAM minimum):
- Mistral-7B-Instruct-v0.2.Q4_K_M.gguf

Lighter option (2 GB RAM):
- Phi-3-mini-4k-instruct.Q4_K_M.gguf

Download from HuggingFace (search "GGUF" + model name).
Save to a folder like `C:\models\`

### Step 2 — Set the environment variable

In PowerShell (temporary, current session only):

```powershell
$env:SAPIENCE_GGUF_MODEL_PATH = "C:\models\mistral-7b-instruct.Q4_K_M.gguf"
```

Or add to your `.env` file (create one in `local_ai/` if needed):

```
SAPIENCE_GGUF_MODEL_PATH=C:\models\mistral-7b-instruct.Q4_K_M.gguf
```

If you skip this step, the app uses pre-written Bengali fallback advice — it still works fine.

## Concurrency Notes (Windows)

### Why --workers 1?

Each uvicorn worker is a separate Python process.
If you have 2+ workers AND a local GGUF model loaded, each worker loads
the model into RAM separately. A 4 GB model × 2 workers = 8 GB RAM used.
On most Windows laptops, `--workers 1` is the right choice.

### asyncio.Semaphore for heavy tasks

The vocal engine and LLM inference are CPU-intensive.
The code uses a `threading.Lock` to prevent two threads from loading
the model at the same time, and a `threading.Semaphore` to allow
only one inference to run at a time.

### FastAPI BackgroundTasks

For operations you want to run after the HTTP response is returned
(e.g., sending a sync notification), use:

```python
from fastapi import BackgroundTasks

@app.post("/some-endpoint")
async def endpoint(background_tasks: BackgroundTasks):
    background_tasks.add_task(my_slow_function, arg1, arg2)
    return {"status": "accepted"}
```

## Troubleshooting

| Problem | Solution |
|---|---|
| `ModuleNotFoundError: pydantic_settings` | Run `pip install pydantic-settings` |
| `SUPABASE_URL validation error` | Check your `.env` file exists and has no quotes around values |
| `librosa not found` | Run `pip install librosa` inside the `local_ai` venv |
| `llama_cpp not found` | Run `pip install llama-cpp-python` or use pre-built wheel |
| `Port 8000 already in use` | Change port: `uvicorn app.main:app --port 8001 --reload` |
| Audio analysis returns "silent" | Speak louder or use a different audio file |
| GGUF model loads slowly | Normal — a 4 GB model takes 10–30 seconds to load |
